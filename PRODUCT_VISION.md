# Legacy Quest: Product Vision
**Gamified Inheritance & Financial Literacy Platform**

---

## The Big Idea

Transform inheritance from a passive wealth transfer into an **active, educational journey** where future generations earn their legacy by demonstrating financial maturity, life skills, and responsibility.

**Tagline:** *"Earn your inheritance. Build your future. Leave a legacy."*

---

## The Three-Layer Platform

### Layer 1: Family Mode (Ages 4-18)
**Current Focus:** Parents teaching kids financial responsibility

**Core Features:**
- Age-tiered progression (4 levels)
- Chore/task gamification
- Points ‚Üí Allowance system
- Financial literacy curriculum
- Achievement unlocks

**Value Prop:** Make allowance meaningful. Teach kids real money skills through gameplay.

---

### Layer 2: Legacy Mode (Multi-generational)
**The Innovation:** Inheritance as an achievement-based game

**Core Concept:**
You (the benefactor) create a "Legacy Quest" for your heirs:
- Set up challenges/milestones tied to REAL inheritance funds
- Achievements unlock actual money from trust accounts
- Educational requirements (courses, certifications, demonstrated skills)
- Life milestones (college graduation, first home, financial stability)
- Character development (volunteer hours, community service)

**Example Quest Chain:**
```
Level 1: Financial Foundations (Unlock: $5,000)
- Complete budgeting course
- Maintain 6-month emergency fund
- Credit score above 700

Level 2: Career & Education (Unlock: $15,000)
- Complete degree/trade certification
- Maintain employment for 12 months
- Create retirement account

Level 3: Wealth Building (Unlock: $50,000)
- Own appreciating asset
- Demonstrate 20% savings rate for 2 years
- Complete investment education

Final Quest: Generational Wealth (Unlock: Remaining estate)
- Create legacy plan for next generation
- Demonstrate sustained financial health
- Complete "Pay it forward" community project
```

**Emotional Hook:** Even after you're gone, you're still teaching them. Your voice, your values, your lessons - encoded in the game.

---

### Layer 3: Legal Integration (Trust Administration)
**The Platform Becomes:** A legally-binding estate planning tool

**Integration Points:**
- Connect to actual trust accounts
- Legal document generation
- Trustee dashboard (verify achievements)
- Financial account verification (Plaid integration)
- Credit score monitoring
- Education verification
- Milestone documentation

**Security & Compliance:**
- Multi-signature verification
- Legal trustee oversight
- Fraud prevention
- Audit trail
- HIPAA/financial data compliance

## Legacy Mode: How It Works

### For the Benefactor (You)

**Setup Phase:**
1. **Create Legacy Account**
   - Link trust/estate account
   - Set total inheritance amount
   - Define beneficiaries (nieces, nephews, grandkids)

2. **Design Quest Chains**
   - Create achievement milestones
   - Assign dollar amounts to each unlock
   - Set educational requirements
   - Define verification criteria

3. **Set Rules**
   - Age requirements (can't unlock until 18, 21, 25, etc.)
   - Timeout rules (must complete within X years)
   - Failure clauses (what happens if they don't achieve)
   - Trustee assignment (who verifies achievements)

4. **Record Your Voice**
   - Video messages for each milestone
   - Letters that unlock with achievements
   - Your story, your wisdom, your values

**Example Setup:**
```
Beneficiary: Nephew Jake (age 12)
Total Inheritance: $100,000

Quest 1: Financial Foundations (Age 18+, Unlock: $10,000)
- Complete budgeting course
- Maintain credit score above 680 for 12 months
- Hold job for 6 months

Quest 2: Education (Age 22+, Unlock: $25,000)
- Complete degree or trade certification
- Maintain GPA above 3.0
- Create retirement account

Quest 3: Stability (Age 25+, Unlock: $30,000)
- 6-month emergency fund
- Own appreciating asset (home/investment)
- Debt-to-income ratio below 30%

Quest 4: Generational Wealth (Age 30+, Unlock: $35,000)
- Create legacy plan for YOUR kids
- Demonstrate 20% savings rate for 3 years
- Complete community service project (100 hours)

Timeout: Must complete all quests by age 40
Failure clause: Unclaimed funds ‚Üí scholarship fund
```

---

### For the Heir (Beneficiary)

**Discovery Phase:**
- Receive notification: "You have a Legacy Quest waiting"
- Video message from benefactor
- See quest chain (but locked until age-eligible)

**Active Play Phase:**
- Progress through Family Mode (ages 4-17)
- Build financial skills
- See future quests as motivation

**Unlock Phase (Age 18+):**
- First quest becomes active
- Complete requirements
- Submit proof to trustee
- Trustee verifies
- Funds released
- Unlock next quest + benefactor message

**Emotional Journey:**
- Each unlock includes a video/letter from you
- Your voice guiding them through life
- Your wisdom at exactly the right moment
- Building skills that will serve them forever

## Technical Architecture

### Database Schema Additions

**New Tables:**

```sql
-- Legacy accounts (benefactors)
CREATE TABLE legacy_accounts (
  id UUID PRIMARY KEY,
  benefactor_user_id UUID REFERENCES users(id),
  total_estate_value DECIMAL(12,2),
  trust_account_info JSONB,
  created_at TIMESTAMP,
  status TEXT -- active, deceased, completed
);

-- Beneficiaries (heirs)
CREATE TABLE beneficiaries (
  id UUID PRIMARY KEY,
  legacy_account_id UUID REFERENCES legacy_accounts(id),
  user_id UUID REFERENCES users(id),
  relationship TEXT, -- nephew, niece, grandchild
  total_allocated DECIMAL(12,2),
  total_unlocked DECIMAL(12,2),
  created_at TIMESTAMP
);

-- Quest chains
CREATE TABLE legacy_quests (
  id UUID PRIMARY KEY,
  legacy_account_id UUID REFERENCES legacy_accounts(id),
  beneficiary_id UUID REFERENCES beneficiaries(id),
  order_number INTEGER,
  title TEXT,
  description TEXT,
  unlock_amount DECIMAL(12,2),
  min_age INTEGER,
  max_age INTEGER,
  requirements JSONB,
  benefactor_message_video_url TEXT,
  benefactor_letter TEXT,
  status TEXT, -- locked, active, pending_verification, completed, failed
  unlocked_at TIMESTAMP
);

-- Quest requirements (granular tracking)
CREATE TABLE quest_requirements (
  id UUID PRIMARY KEY,
  quest_id UUID REFERENCES legacy_quests(id),
  requirement_type TEXT, -- education, credit_score, employment, savings, asset
  criteria JSONB,
  verification_method TEXT, -- manual, automatic, trustee
  status TEXT, -- pending, verified, rejected
  submitted_at TIMESTAMP,
  verified_at TIMESTAMP,
  verified_by UUID REFERENCES users(id)
);

-- Trustee assignments
CREATE TABLE trustees (
  id UUID PRIMARY KEY,
  legacy_account_id UUID REFERENCES legacy_accounts(id),
  user_id UUID REFERENCES users(id),
  permissions JSONB,
  added_at TIMESTAMP
);

-- Unlocked funds (distribution history)
CREATE TABLE fund_distributions (
  id UUID PRIMARY KEY,
  beneficiary_id UUID REFERENCES beneficiaries(id),
  quest_id UUID REFERENCES legacy_quests(id),
  amount DECIMAL(12,2),
  distribution_method TEXT, -- direct_deposit, trust_transfer, check
  account_info JSONB,
  distributed_at TIMESTAMP,
  verified_by UUID REFERENCES users(id)
);
```

## Legal Considerations

**Trust Structure:**
- Legacy accounts must be backed by actual trusts
- Platform acts as administrator, not fiduciary
- Trustee oversight required for fund releases
- State-specific trust law compliance

**Liability:**
- Clear terms: Platform facilitates, doesn't guarantee
- Benefactor chooses trustees (we don't decide distributions)
- All distributions require multi-signature verification
- Dispute resolution process

**Data Security:**
- Financial data: PCI-DSS compliant
- Personal data: GDPR/CCPA compliant
- Estate documents: Encrypted at rest
- Multi-factor authentication required
- Audit logs for all transactions
